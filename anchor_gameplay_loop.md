# 《船锚》核心循环与动作化验证草案

## 1. 核心概念

玩家经营一艘小帆船，在镇子上用有限资金购买材料，手动拼接一个“会工作的船锚机关”，然后装载货物出海贸易。航线上必定遭遇风暴，玩家下锚后，系统不只是计算重量和摩擦，而是把玩家拼出的结构转化成一段可见的动作验证：锚如何入水、翻身、展开、咬地、缓冲、撑住船。

一句话玩法：

> 买材料，亲手拼一个会动作的锚机关；风暴来了，看它能不能靠可见动作把船稳住。

当前设计重点：

1. **玩家如何手动拼接一个能工作的锚结构。**
2. **风暴验证中，材料功能必须通过动作表现出来。**
3. **成败原因必须能被玩家一眼看懂。**

本方向的核心不是“重物 + 链条 + 摩擦系数”，而是“被船拉动后会发生动作的机械结构”。玩家不需要理解公式，但要能看见：钩爪为什么抓住，阻力伞为什么减速，海绵为什么救了连接，船绳挂点为什么把整个锚拉翻。

## 2. 设计原则：可见因果

参考 Horsey Game 的直观性：

> 腿往后转 → 膝盖顶地 → 身体被推向前。

玩家能直接看到动作和结果之间的关系。船锚玩法也必须建立类似链条：

> 船绳拉动 → 锚结构翻身 / 展开 / 咬地 / 缓冲 → 船被稳定。

不推荐的表达：

- 铁块重，所以减速。
- 摩擦高，所以停船。
- 抓地力数值够，所以成功。

推荐的表达：

- 钩爪被拉着翻下去，插进岩缝。
- 广告牌被水流撑开，像阻力伞一样拖住船。
- 海绵被第一下冲击压扁，连接没有立刻断。
- 木梁横着顶住海床，结构不再继续翻滚。
- 船绳挂点太偏，整个锚被拉成反方向，钩子朝天滑走。

## 3. 最终判定标准

玩家最容易理解的最终判定只有一个：

> 风暴期间，船在限定时间内没有被吹翻，也没有被冲走。

现阶段不把“船体耐久”作为主判定。船的失败主要来自两个可见结果：

- 船体倾斜超过翻船阈值。
- 船被风暴推离安全区域。

成功分三档：

- 稳定通过：锚的关键动作进入工作状态，船体倾斜小，货损低。
- 惊险通过：锚有部分动作成功，但抓地浅、缓冲不足或阻力不稳，船没翻但货损较高。
- 失败：锚没有进入工作状态，或动作方向错误，船翻了 / 被冲走 / 货物全损。

## 4. 单局核心循环

推荐将一局组织成以下流程：

> 镇子筹备 → 买材料 → 手动拼锚 → 指定船绳挂点 → 装货出海 → 风暴动作验证 → 结算盈亏 → 回镇升级

### 4.1 镇子筹备

玩家每轮开始时回到镇子，获得当前资金和市场信息。

镇子提供：

- 材料商店：购买钩爪、木梁、铁块、广告牌、海绵、轮胎、水母等材料。
- 工坊：手动拼接船锚，调整材料位置、角度和连接。
- 货栈：选择本轮要运的货物。
- 工程所：后续解锁改锚、测锚、炼锚、驯锚、万物锚化等工程手段。

玩家核心问题：

> 这笔钱是买更赚钱的货，还是买能做出关键动作的材料？

### 4.2 买材料

材料不是抽象数值卡牌，而是后续拼装场景中的实体零件。每个材料最重要的不是属性高低，而是它在验证阶段能表现出什么动作。

材料至少包含：

- 外形：长条、块状、片状、钩状、柔性、不规则。
- 连接点：孔、边、角、软组织、可缠绕区域。
- 工作动作：咬地、展开、缓冲、撑地、配重、黏附、回弹等。
- 姿态需求：必须朝下、必须横放、必须被拉开、必须先触底等。
- 风险：装反、挂点错误、被扯掉、回弹、变形过度、挡住其他动作等。
- 价格：影响玩家是否舍得买。

材料示例：

| 材料 | 主要工作动作 | 玩家能看到的表现 | 失败表现 |
| --- | --- | --- | --- |
| 废铁钩 | 咬地 | 被船绳拉动后翻下去，钩尖插入海床或岩缝 | 钩尖朝天滑走，或角度太浅只刮出痕迹 |
| 广告牌 | 开伞 / 阻水 | 被水流撑开，面板鼓起，拖出明显水痕 | 侧着入水，没展开；连接点被扯掉 |
| 木梁 | 撑地 / 力臂 | 横着顶住海床，阻止锚继续翻滚 | 竖着插空，或被拉成无效角度 |
| 铁块 | 配重 / 翻身 | 压住某一侧，让钩爪自动翻到工作方向 | 太靠后，把钩子压反；太重导致结构拖不动 |
| 海绵 | 缓冲 | 第一波拉力来时被压扁，连接线从红色回落 | 太软，拉力被吃掉，钩爪没有咬进去 |
| 轮胎 | 回弹 / 套接 | 被拉扁后回弹，吸收冲击或套住石头 | 回弹把结构弹翻 |
| 水母 | 黏附 / 变形 | 被拖动时拉长并黏住海床或材料 | 黏错位置，拖住关键运动 |
| 怪石 | 卡缝 / 配重 | 不规则边缘卡在海床凸起处 | 圆滑滚走，或挡住钩爪展开 |

## 5. 手动拼锚

这是现阶段的主玩法。

玩家在工坊中看到拼装台和购买来的实体材料。玩家需要亲自完成以下操作：

1. 拖拽材料到拼装台。
2. 旋转材料，调整钩爪、支架、阻力面板的朝向。
3. 选择材料上的连接点。
4. 使用绳子、链条、包裹等方式连接材料。
5. 指定船绳挂点，决定船会从哪里拉动整个锚结构。
6. 带着这个结构出海验证。

拼装不是“放进槽位就生效”，而是生成一个动作结构：

- 材料是会移动、翻滚、受拉的实体。
- 连接点决定拉力从哪里进入结构。
- 船绳挂点决定结构被拉动后的第一动作。
- 材料之间的距离和角度决定是否能翻身、展开、咬地。
- 柔性材料会变形，硬质材料会撑住或断开。

### 5.1 船绳挂点

船绳挂点是核心决策，不应由系统自动选择。

玩家必须手动指定船和锚的连接点。这个点决定：

- 船拉动锚时，锚先向哪个方向旋转。
- 钩爪是被拉下去，还是被拉到朝天。
- 阻力伞是被水流打开，还是被折起来。
- 缓冲件是否处在受力路径上。
- 支架是否能顶住海床。

同一个锚结构，船绳挂在不同位置，应产生明显不同的验证表现。

## 6. 连接操作规则

连接件不是纯强度加成，而是动作传递方式。

| 连接件 | 动作意义 | 优点 | 风险 |
| --- | --- | --- | --- |
| 绳子 | 延迟拉动 / 缓冲 | 有弹性，能给钩爪翻身时间 | 承拉低，可能被瞬间拉断 |
| 链条 | 直接传力 | 拉力清晰，适合强抓地件 | 太硬，容易把结构直接拽翻 |
| 包裹 | 固定软材料 | 适合海绵、水母、轮胎 | 会变形，可能让角度失真 |
| 卡扣 | 短距离硬连接 | 保持机关角度 | 连接点要求高，错位容易失效 |
| 穿刺 | 让软材料参与结构 | 可制造奇怪复合件 | 可能撕裂软材料 |

连接需要可视化：

- 拉伸时变黄。
- 过载时变红。
- 断裂时弹开，并留下断裂日志。
- 柔性连接应明显拉长。
- 硬连接应明显把拉力快速传给下一个材料。

## 7. 装货出海

玩家选择本轮贸易货物。货物越重，潜在利润越高，但风暴中更难稳定。

货物设计原则：

- 轻货：利润低，给玩家更多试验空间。
- 重货：利润高，但船更容易倾斜，被锚拉住时也更容易晃。
- 易碎货：船没翻也可能因为摇晃造成货损。
- 压舱货：利润一般，但降低翻船风险。

货物重量不应遮蔽锚的动作反馈。它主要改变风暴压力，而不是替代锚机关的表现。

## 8. 风暴动作验证

这是现阶段的第二个主玩法重点。

风暴阶段必须回答一个问题：

> 玩家刚才拼的东西，具体是靠什么动作把船稳住的？

验证分为三段：

### 8.1 入水落地：姿态是否对

锚进入水中后，下沉、翻滚并接触海床。

需要展示：

- 哪个材料先触底。
- 钩爪是否朝下。
- 阻力板是否处于能展开的方向。
- 配重是否让结构自动翻到可工作姿态。
- 未连接材料是否分离。

典型失败：

- 钩子装反，触底后钩尖朝天。
- 配重位置错误，整个锚平躺。
- 广告牌挡住钩爪，关键动作无法发生。

### 8.2 船绳拉动：结构是否进入工作状态

船绳开始受力后，结构应出现明确动作。

需要展示：

- 钩爪被拉动后翻下去并咬进海床。
- 阻力伞被水流撑开。
- 海绵 / 轮胎被压缩，吸收第一波冲击。
- 木梁被拉到横向，顶住海床。
- 连接线拉伸、变色、断裂。
- 船绳挂点如何改变整个结构的旋转方向。

这一段是最接近 Horsey Game 的部分：  
玩家要看到“它动了，所以它生效了”。

### 8.3 风暴持续：动作是否稳定

锚进入工作状态后，还要撑过一段时间。

需要展示：

- 钩爪插入深度或刮痕。
- 阻力伞持续鼓起或被撕开。
- 缓冲件反复压缩回弹。
- 支架是否滑脱。
- 船体倾斜是否被压回安全范围。
- 货物是否因为摇晃造成损失。

典型失败：

- 钩爪咬得太浅，被慢慢拖出来。
- 阻力伞打开了，但连接点被撕掉。
- 缓冲件太软，钩爪迟迟咬不进去。
- 船绳挂点太偏，锚不断翻滚，无法进入工作姿态。

## 9. 可见反馈与失败解释

玩家失败时，系统应该用画面和短句说明原因。

推荐反馈：

- “钩爪朝天，无法咬地。”
- “船绳挂点太高，锚被拉翻了。”
- “阻力伞没有展开，被水流卷走。”
- “海绵吸收了冲击，但拉力没有传到钩爪。”
- “木梁没有横撑，结构继续滚动。”
- “钩爪咬地成功，但连接件断裂。”
- “锚稳定了，但货物太重，船仍然倾斜过大。”

反馈应优先描述动作问题，而不是数值问题。

不推荐：

- “抓地力不足。”
- “摩擦系数不够。”
- “稳定值低于阈值。”

推荐：

- “钩子只刮出浅痕，没有插进去。”
- “广告牌侧着拖行，没有被水撑开。”
- “船绳第一下把锚头拉反了。”

## 10. 结算盈亏

风暴结束后，根据结果结算。

收入：

- 货物基础利润。
- 准时抵达奖励。
- 高风险航线奖励。
- 低货损奖励。

支出：

- 材料成本。
- 货物损失。
- 锚损坏或遗失。
- 债务利息或保险费用。

现阶段结算公式倾向：

> 本轮净收益 = 贸易收入 - 材料成本 - 货损 - 锚损耗

玩家成功的爽点是：

> 我亲手拼出来的怪东西，真的靠一个看得懂的动作救了这一船货。

## 11. 现阶段实现重点

### 11.1 手动拼接优先级最高

最低可验证版本应包含：

- 可拖拽材料。
- 可旋转材料。
- 可选择连接点。
- 可创建连接件。
- 可删除连接件。
- 可指定船绳挂点。
- 未连接材料在验证阶段会分离。
- 连接件有可见拉伸、变红、断裂状态。

### 11.2 动作化验证优先级第二

风暴验证必须能看出“玩家刚才怎么拼”对动作结果的影响。

最低可验证版本应包含：

- 钩爪类材料：能翻身、刮地、咬地。
- 阻力板类材料：能被水流撑开或失败折叠。
- 缓冲类材料：能压缩、回弹、延迟拉力。
- 支架类材料：能顶住海床或滑脱。
- 船绳挂点：明显影响锚的旋转方向。
- 船体倾斜：反映锚是否真的稳住船。
- 货损：反映通过时是否稳定。

### 11.3 数值退到幕后

数值仍然需要存在，但不能成为玩家理解的第一层。

内部可以有：

- 质量。
- 拉力。
- 弹性。
- 抓地阈值。
- 水阻。
- 连接强度。

玩家看到的应该是：

- 翻身。
- 展开。
- 咬住。
- 压缩。
- 撑住。
- 断裂。
- 滑走。

## 12. 长线成长结构

长线成长不只是数值变大，而是逐步解锁新的“造锚动作学”。

推荐递进：

> 普通材料 → 手动拼锚 → 改锚 → 测锚 → 炼锚 → 驯锚 → 万物锚化

### 12.1 普通材料

早期玩家只接触现实材料，目标是建立基础认知：

- 钩爪负责咬地。
- 面板负责开伞阻水。
- 软材料负责缓冲。
- 长条材料负责撑地和力臂。
- 配重材料负责改变落地姿态。
- 船绳挂点负责决定第一动作。

### 12.2 改锚

玩家可以加工材料，让材料产生更明确的动作。

| 改造 | 影响拼装 | 影响验证 |
| --- | --- | --- |
| 削尖 | 增加尖端方向要求 | 钩爪更容易插入，但装反更废 |
| 打孔 | 新增连接点 | 改变船绳拉动方向 |
| 压弯 | 改变材料曲率 | 更容易形成钩爪或支架 |
| 压扁 | 增加面板面积 | 更容易开伞，但更易被扯掉 |
| 加弹簧 | 增加缓冲段 | 第一波拉力更柔和 |
| 加铰链 | 允许展开动作 | 可做折叠伞、折叠爪 |

### 12.3 测锚

解锁电脑模拟后，玩家可以在出海前预测动作风险。

模拟项目：

- 入水姿态预测。
- 船绳拉动方向预测。
- 钩爪是否朝向海床。
- 阻力伞展开概率。
- 缓冲件是否处于受力路径上。
- 第一断裂点预测。
- 翻船风险预测。

模拟不是替代风暴验证，而是帮助玩家理解自己的拼装结构。

### 12.4 炼锚

玩家开始融合材料，制造复合锚材。

| 融合材料 | 结果 | 动作表现 |
| --- | --- | --- |
| 海绵 + 铁粉 | 吸水配重垫 | 入水后变沉，同时压缩缓冲 |
| 水母 + 绳子 | 黏性活绳 | 拉长后黏住海床或材料 |
| 鱼骨 + 轮胎 | 弹性倒刺环 | 被拉扁后倒刺咬地 |
| 木梁 + 贝壳 | 硬壳支架 | 横撑更稳，但角度要求高 |
| 广告牌 + 铰链 | 折叠阻力伞 | 被水流打开，失败时会拍回去 |

### 12.5 驯锚

生物材料进入后，锚可以有主动动作。

- 水母主动黏附海床。
- 章鱼缠住礁石或锚主体。
- 螃蟹夹住岩缝，成为活抓地件。
- 鱼群反向游动，提供持续拖拽。

生物材料也必须遵守可见因果：玩家要看见它做了什么，而不是只得到数值加成。

### 12.6 万物锚化

后期解锁荒诞工程后，玩家可以把各种对象定义成锚材料、连接件或规则约束。

- 车：轮胎回弹、车架撑地、车门开伞。
- 房子：大型阻力结构，窗框可卡住海床。
- 人：行为约束，主动拉、抱、踹、撑。
- 广告牌：水下阻力伞。
- 时间：短暂固定某个成功动作姿态。
- 债务：越拖越重的概念约束。

万物锚化必须仍然服务同一目标：

> 改变玩家可拼装的材料、连接方式或风暴验证中的可见动作。

## 13. 核心风险与取舍

### 13.1 资金取舍

玩家每轮预算有限。

- 多买动作材料：锚更容易进入工作状态，但货物利润降低。
- 多买货物：利润更高，但对锚动作稳定性要求更高。
- 买便宜材料：能试奇怪结构，但关键动作可能缺件。
- 买贵材料：动作更明确，但失败成本更高。

### 13.2 货物重量取舍

货物不是单纯奖励，也会改变航行压力。

- 轻载：收益低，但验证容错高。
- 重载：收益高，但需要更稳定的抓地和缓冲。
- 偏载：更容易倾斜，迫使玩家做更可靠的锚。
- 易碎货：船没翻也可能因为晃动产生货损。

### 13.3 锚结构取舍

好锚不是越重越好，也不是连接越强越好。

- 钩爪多：抓地机会多，但容易互相挡住。
- 面板大：阻力强，但容易被扯掉。
- 缓冲多：第一下不容易断，但可能让钩爪咬不进去。
- 支架长：力臂强，但入水时容易翻滚。
- 配重大：容易翻身，但可能把动作压死。
- 船绳挂点靠前：拉动直接，但可能把锚掀翻。
- 船绳挂点靠后：容易拖动整体，但可能无法让钩爪下压。

## 14. 首版可实现范围

一阶段先实现：

- 镇子界面：显示资金、材料商店、货物列表。
- 购买材料和货物。
- 2D 手动拼锚：拖拽、旋转、连接点、连接件、删除连接。
- 手动指定船绳挂点。
- 风暴动作验证：入水、触底、拉动、翻身、展开、咬地、缓冲、撑地。
- 可视化失败原因：钩爪朝向错误、阻力伞未展开、缓冲无效、支架滑脱、连接断裂、船翻。
- 盈亏结算：收入、材料成本、货损、净收益。
- 稳定通过、惊险通过、失败三类结果。

一阶段暂不做：

- 完整航海地图。
- 复杂流体模拟。
- 复杂船员系统。
- 深度生物 AI。
- 万物锚化。
- 多港口长期经济系统。
- 深度工程所升级树。

## 15. 后续扩展方向

- 工程所：逐步解锁改锚、测锚、炼锚、驯锚、万物锚化。
- 航线选择：高利润航线风暴更强，对锚动作要求更高。
- 保险和贷款：让玩家在经济层面继续冒险。
- 货物摆放：重心管理影响翻船概率。
- 锚蓝图收藏：保存成功设计，后续可复用或改良。
- 每日挑战：相同市场和风暴，比拼最高净收益。

## 16. 设计重点

新版核心循环应当让玩家每轮都产生三个问题：

1. 我这笔钱应该买货，还是买能产生关键动作的材料？
2. 我该怎么把这些材料连接成一个会翻身、展开、咬地、缓冲的锚机关？
3. 船绳一拉，我的结构会先做出什么动作？

这个方向的重点不是让玩家点几个槽位拿结果，也不是让玩家相信隐藏数值。玩家要亲手制造一个结构，然后看这个结构在风暴中做出动作，并因为这些动作成功或失败。

> 我不是为了怪而怪，我是在用越来越离谱的材料和工程手段，亲手拼出一个能靠可见动作保护一船货的锚。
