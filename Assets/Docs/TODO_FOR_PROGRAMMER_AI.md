# 给程序 AI 的任务清单

## 禁止先做

- 不要启动服务器。
- 不要做网络功能。
- 不要先做存档。
- 不要先做完整美术。
- 不要先做复杂真实流体。
- 不要把连接规则写成重叠面积。
- 不要把开始菜单、开局、拼装、模拟、结算塞进同一屏。

## 第一优先级：界面流

1. 建 `Prototype.unity`。
2. 建 `ViewFlowController`。
3. 建 6 个 UI 根节点或 Prefab：
   - MainMenuView
   - SettingsView
   - StartBriefingView
   - BuildView
   - SimulationView
   - ResultView
4. 实现界面跳转：MainMenu -> Briefing -> Build -> Simulation -> Result。

## 第二优先级：BuildView

1. 放 5 个测试材料。
2. 做拖拽、旋转、翻转。
3. 做边框贴合检测。
4. 做连接生成和连接图。
5. 做提交风险提示，不阻挡下锚。

## 第三优先级：SimulationView

1. 入水阶段：底部短时大冲力。
2. 下沉阶段：重力、浮力、水流横向力、船锚绳拉力。
3. 连接位移、滑移、贴合长度损伤。
4. 部件脱落。
5. 触底阶段：摩擦、支撑、抓地。
6. 船向危险区漂移。
7. 船锚绳反拉船。

## 第四优先级：ResultView

1. 成功 / 勉强成功 / 失败。
2. 船是否进入危险区。
3. 船锚损伤。
4. 失败原因。
5. 重开或下一局。

## 最小可玩验收

- 能看到主菜单和四个局内界面流程。
- 拼装时只专注拼锚。
- 模拟时只专注看船是否被拉住。
- 结算时能解释失败原因。
