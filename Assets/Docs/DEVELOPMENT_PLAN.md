# Development Plan 开发模块计划

## 开发总目标

先做一个可跑的 Unity 2D 原型，验证完整闭环：

```text
MainMenuView
  -> StartBriefingView
  -> BuildView
  -> SimulationView
  -> ResultView
```

一句话验收：玩家能从开始界面进入一局，看到材料，拼出一个临时船锚，下锚后经历入水、下沉、触底；船被横向力推向危险区，船锚通过绳子拉住船，最后根据船是否进入危险区结算。

---

## 模块 1：流程与界面

优先级：最高。

### 内容

- `MainMenuView`：开始 / 设置 / 退出。
- `SettingsView`：音量、画面质量占位、调试显示开关、返回。
- `StartBriefingView`：展示船重、海况、水深、海底、危险区距离、目标时间、材料。
- `BuildView`：拼装界面。
- `SimulationView`：下锚模拟界面。
- `ResultView`：结算界面。
- `ViewFlowController`：界面跳转。
- `RoundController`：一局流程控制。

### 验收

- 能从开始界面一路走到结算界面。
- 每个界面职责清楚，不混在一屏。
- 设置界面能从主菜单打开并返回。

---

## 模块 2：拼装系统

优先级：最高。

### 内容

- 材料生成。
- 拖拽。
- 旋转。
- 翻转。
- 选择船锚绳绑点。
- 拼装边框数据。
- 边框贴合检测。
- 连接生成。
- 连接图。
- 主体连通检测。
- 风险提示。

### 关键规则

- 不用重叠面积。
- 用边框贴合长度。
- 判定允许冗余距离。
- 提交不硬拦，只提示风险。

### 验收

- 玩家能拼 3-5 个物品。
- 能看到连接高亮。
- 未连通材料会提示。
- 绑点风险会提示。
- 点击下锚能进入模拟。

---

## 模块 3：连接与损伤系统

优先级：最高。

### 内容

- `AttachJoint`。
- 当前贴合长度。
- 当前连接强度。
- 相对位移。
- 拉伸状态。
- 滑移状态。
- 断开状态。
- `JointDamageSystem`。

### 关键规则

```text
受力
  -> 相对位移
  -> 超过弹性位移
  -> 滑移
  -> 贴合长度下降
  -> 达到阈值才断开
```

### 验收

- 下沉阶段不会一受力就散架。
- 入水阶段弱连接可以被冲开。
- 能看到连接状态变化。

---

## 模块 4：三阶段模拟

优先级：高。

### 内容

- `WaterEntryStage`。
- `SinkStage`。
- `SeabedStage`。
- `ForceModel`。
- `PieceMotionState`。

### 阶段规则

- 入水：底部短时大冲力，可冲开弱连接。
- 下沉：重力、浮力、水阻、水流横向力、船锚绳拉力。
- 触底：水流降低，加入海底摩擦、支撑、抓地。

### 验收

- 入水有明显冲击。
- 下沉有持续拉扯。
- 触底后能产生抗拖拽效果。

---

## 模块 5：船、绳子与危险区

优先级：高。

### 内容

- `ShipDriftSimulator`。
- `RopeForceSystem`。
- 危险区边界。
- 稳船倒计时。
- 船位置。
- 船漂移速度。
- 船锚反拉力。

### 成功逻辑

```text
船没有进入危险区
且 坚持到目标时间
且 船锚绳未断
且 绑点主体未断
```

### 验收

- 船会被横向力推向危险区。
- 船锚能通过绳子减缓船漂移。
- 船进危险区失败。
- 倒计时结束未进危险区成功。

---

## 模块 6：数据配置

优先级：中。

### 内容

- `MaterialConfig`。
- `AnchorPieceConfig`。
- `AttachConfig`。
- `StageConfig`。
- `LevelConfig`。

### 优先字段

- 材料重量。
- 密度。
- 粘合系数。
- 水阻。
- 摩擦。
- 抓地。
- 贴合冗余距离。
- 最小贴合长度。
- 入水冲击。
- 水流横向力。
- 危险区距离。
- 目标时间。

### 验收

- 不改代码也能调材料和关卡参数。
- 至少能配置 5 个测试材料。

---

## 模块 7：Debug 与验证工具

优先级：中，但不要拖到最后。

### 内容

- 显示拼装边框。
- 显示连接线。
- 显示连接状态：稳定 / 拉伸 / 滑移 / 断开。
- 显示力方向。
- 显示船距离危险区。
- 显示失败原因。

### 验收

- 程序能看出为什么断。
- 策划能调参数。
- 玩家失败原因能归因。

---

## 推荐开发顺序

```text
1. 流程与界面空壳
2. 拼装拖拽 / 旋转 / 翻转
3. 边框贴合连接
4. 风险提示
5. 入水冲击
6. 下沉受力 + 连接损伤
7. 触底摩擦 / 支撑
8. 船漂移 + 绳子反拉 + 危险区
9. 结算
10. Debug 可视化和数值调参
```

---

## 第一版最小 Demo 验收

- 能从 `MainMenuView` 点击开始进入 `StartBriefingView`。
- 能从 `StartBriefingView` 进入 `BuildView`。
- 能在 `BuildView` 拼 3-5 个物品。
- 能看到边框贴合连接高亮。
- 能看到风险提示，但不会被硬拦下锚。
- 能进入 `SimulationView`。
- 入水阶段能冲开弱连接。
- 下沉阶段连接先滑移再断。
- 触底后船锚能减缓船漂移。
- 船未进危险区成功，进危险区失败。
- 能进入 `ResultView` 查看成功/失败和原因。
