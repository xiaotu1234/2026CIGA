# 断锚求生 Unity 原型说明

本目录用于验证一个 2D 拼装船锚玩法原型。

## 核心目标

船持续受到横向外力，向远处危险区漂移。玩家用随机材料拼出临时船锚，下锚后船锚通过绳子反拉船。船在目标时间内没有进入危险区，则本局成功。

## 界面流

不要把所有内容塞进一屏。原型按 5 个主界面拆分：

1. MainMenuView：开始界面，包含开始、设置、退出。
2. StartBriefingView：开局局势 / 捞取材料界面。
3. BuildView：拼装船锚界面。
4. SimulationView：下锚模拟界面。
5. ResultView：结算反馈界面。

界面流程：

```text
StartBriefingView
  -> BuildView
  -> SimulationView
  -> ResultView
```

## 一期只验证这些

1. 2D 物品拖拽、旋转、翻转。
2. 物品通过 2D 拼装边框贴合生成连接。
3. 贴合判定有冗余空间，不要求碰撞箱严格接触。
4. 物品有显示图形、拼装边框、实际碰撞箱三层数据。
5. 提交下锚不做硬性阻挡，只做风险提示。
6. 下锚后只分入水、下沉、触底三个阶段。
7. 材料受到力后先产生位移、滑移和贴合长度损伤，再断开。
8. 入水阶段允许弱连接被短时大冲力直接冲开。
9. 下沉阶段不要一受力就散架，应表现持续拉扯。
10. 触底阶段加入海底摩擦、支撑、抓地。
11. 船受到横向力漂移，船锚通过绳子反拉船，船未进危险区则成功。

## 重要规则

- 不使用重叠面积作为连接依据。
- 使用边框贴合长度作为连接依据。
- 拼装界面只专注拼锚，不实时展示完整船漂移模拟。
- 模拟界面只专注看船锚是否救得住船，不允许继续编辑。
- 结算界面必须解释失败原因。

## 推荐先做的材料

- IronBlock：铁块，重，适合主体。
- WoodPlank：木板，边框长，易连接，轻。
- Hook：铁钩，适合触底抓地。
- Stone：石块，重，不规则。
- Tire：轮胎，水阻大，有缓冲。
